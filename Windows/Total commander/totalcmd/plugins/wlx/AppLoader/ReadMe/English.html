Description
This plugin allows you to open chosen files using the associated applications in cases where it is possible to use the internal viewer (Lister).
The main feature of this plugin is that you can open the file in any application of these modal windows Total Commander as “Find Files” and “Synchronize directories” as well as in the Overwrite dialog, which can not be done by any other means.
No less pleasing is provided feature to run associated application by means hotkey for an already opened file in the Lister.
Also plugin can work with selected files, has a flexible configuration of execute conditions, and feature to disable using the Scroll Lock key.

Installation
Automatic
  1. Enter in the archive with the plugin (Ctrl+PgDn) and Total Commander will ask you to install it
  2. Follow the instructions of Total Commander
Manual
  1. Unpack the contents of the archive to a separate directory
  2. Install plugin:
    • { Configuration → Options... → Plugins → Lister plugins → Configure }
    • Add AppLoader.wlx (AppLoader.wlx64) to the plugins list
  3. Move AppLoader on the top of the plugins list
Examples of convenient use
  • Use in modal windows.
    You no longer need to run another copy of Total Commander, if the need arises, for example, open a document in Microsoft Word or play video with the handy player in “Find Files”, “Synchronize directories” or Overwrite dialog. Just configure associations for convenient applications.
  • Use to open files from FTP-servers and WFX-plugins.
    You no longer need to pre-copy file on the local disk to view or extract the contents. Just configure associations for convenient archiver.
  • Working with selected files.
    You need to open a group of files in separate tabs of text editor or print them in one motion? Just select the appropriate application and correctly configure the associations and command line parameters.
  • Association application with folders.
    Want to open in a player the entire season of your favorite series? Make sure that your player is able to work with folders and set associations.
  • Launching applications for already opened file in Lister.
    You have viewed the file in Lister and realized that you need to edit it? Just assign hotkey to launch the associated application and do not forget to use it :)
  • Copy searched text to clipboard.
    You already have appreciated all plugin features and use it to the full in “Find Files” dialog? Then this function is for you. If you're searched for files with option “Find text” plugin will automatically copy the search text to the clipboard, and you can paste it later in the search dialog of associated application.
Limitations
  • Operability of all the features may be guaranteed only for final versions of Total Commander 8.0, 8.01, бета-версии 8.5b15. For previous versions there is a significant probability of losing functional, for following versions the functional loss is also not excluded.
  • Launch the associated application will be performed outside the Lister window (both in normal and in the quick view).
  • When plugin executed from archive/FTP-server/WFX-plugin, due to the specific behavior of Total Commander, selection can not be processed completely and will be processed only the first selected file.
Configuration
The plugin can be configured by editing the file AppLoader.ini which should be in plugin installation folder. Create this file manually or copy from folder IniSample. If this file is not found when plugin is loading into memory, it will be copied automatically from the folder IniSample.
The following are the keys to the configuration file with descriptions. The key value specified after the '=' is the default value, ie value that will be used if key is not exists in the file.

1: section Extensions in Total Commander configuration file;

Notes on file names: If name of application is value but not a key (i.e. placed on the right of the '='), in its name:
  • can be used environment variables such as %COMMANDER_PATH%
  • can be used relative paths
  • are allowed to specify file name without path, if executive application file is located in plugin directory, system directory, Windows directory or in directory which defined in system variable PATH
Notes on file extensions: Besides the usual extensions also can be used the special extensions:
  • extension :dir for folders (for example, htm,html,:dir,xml=C:\Program Files\Opera\opera.exe)
  • empty extension (for example, txt,,ini=notepad, ,txt,ini=notepad or =notepad, but not txt,ini,=notepad)
Notes on command line parameters:
  • When using file names in the parameters are actual notes above.
  • Use special symbol %1 in string of command line parameters (keys of section Editor, key DefaultEditor_params of section Options) to define in which position name(s) of file(s) will placed.
    Sample: For file <file>, opening in application <application>, is defined the following string of command line parameters: <parameters_1> %1 <parameters_2>.
    The result will be executing the command: <application> <parameters_1> <file> <parameters_2>.



Explanations on work keys ParamsMode and ParamsByExtMode.
When executed plugin for a single file (that is, from modal dialogs, Lister or for file under cursor in Total Commander file window) will be started only one application. In this case, if ParamsMode=1, then for each of the values ​​ParamsByExtMode passing the command line parameters to application will be performed the same way – as an exception in the case of a single file.
If the executing occurs from file window of Total Commander, while more than one file is selected, there are several variants of forming commands to run one or more applications. This example illustrates how the plugin works at different values of keys ParamsMode and ParamsByExtMode.
In active panel in the directory D:\files\ selected the following files:
file0.ext0
file1.ext0
file0.ext1
file1.ext1
file0.ext2
In section Editor presents the following strings:
[Editor]
ext1=C:\folder\dir\app0.exe
ext0,ext2=D:\path\app1.exe
ext1_params=-p1 -p2
ext0,ext2_params=/key0
ext0_params=/key1
C:\folder\dir\app0.exe=-p3
D:\path\app1.exe=/key3
Applications will be run this way:
1. If ParamsMode=0
"D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" "D:\files\file0.ext2" /key3
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" "D:\files\file1.ext1" -p3
2. If ParamsMode=1, ParamsByExtMode=0
"D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" "D:\files\file0.ext2" /key1
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" "D:\files\file1.ext1" -p1 -p2
3. If ParamsMode=1, ParamsByExtMode=1
"D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" /key1
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" "D:\files\file1.ext1" -p1 -p2
"D:\path\app1.exe" "D:\files\file0.ext2" /key0
4. If ParamsMode=1, ParamsByExtMode=2
"D:\path\app1.exe" "D:\files\file0.ext0" /key1
"D:\path\app1.exe" "D:\files\file1.ext0" /key1
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" -p1 -p2
"C:\folder\dir\app0.exe" "D:\files\file1.ext1" -p1 -p2
"D:\path\app1.exe" "D:\files\file0.ext2" /key0
Keep in mind, if the application is run with multiple files (for example, "D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" "D:\files\file0.ext2"), all of them will be opened in this application only if it is multiple document application. Otherwise, the application will only open the first file.

Configuration of Total Commander
AppLoader behavior in some cases may depend on the Total Commander settings, made ​​through its configuration dialog or configuration file.

Configuration dialog
Configuration file
Description
Page “Edit/View”, options in group “Viewer for F3” Section [Configuration], key Viewertype Used by plugin to determine the behavior in the presence of selection in the Total Commander file panels.
Page “Edit/View”, button “Associations...” Section [Extensions] Can be used by plugin for replacement section Editor of own configuration file.
  Section [Configuration], key GetTextMode Used by plugin to determine the format in which Total Commander will return a list of selected files. When the key value is 0, plugin will not work with the selected files whose names contain spaces.

Full description of these settings can be found in the Total Commander help file.
