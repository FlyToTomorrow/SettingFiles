Описание
Плагин позволяет открывать выбранные файлы с помощью ассоциированных приложений в тех случаях, когда есть возможность воспользоваться внутренним просмотрщиком (Lister).
Главной особенностью этого плагина является возможность открыть файл в любом приложении из таких модальных окон Total Commander как «Поиск файлов» и «Синхронизация каталогов», а также в Диалоге подтверждения перезаписи, чего нельзя осуществить никакими другими средствами.
Не менее приятной является предоставляемая плагином возможность запуска ассоциированного приложения по горячей клавише для уже открытого во внутреннем просмотрщике файла.
Кроме того плагин может работать с выделенными файлами, имеет гибкую настройку условий срабатывания, а также функцию отключения клавишей Scroll Lock.

Установка
Автоматическая
  1. Войдите в архив с плагином (Ctrl+PgDn) и Total Commander предложит вам установить его
  2. Следуйте указаниям Total Commander
Ручная
  1. Распакуйте содержимое архива в отдельный каталог
  2. Подключите плагин:
    • { Конфигурация → Настройка... → Плагины → Плагины внутреннего просмотрщика → Настройка }
    • Добавьте AppLoader.wlx (AppLoader.wlx64) в список плагинов
  3. Поместите AppLoader на первое место в списке плагинов
Примеры удобного использования
  • Использование в модальных окнах.
    Больше не нужно запускать другую копию Total Commander, если возникает необходимость, например, открыть документ в Microsoft Word или воспроизвести видеоролик с помощью удобного плеера в диалогах «Поиск файлов», «Синхронизация каталогов» или Диалоге подтверждения перезаписи. Просто настройте ассоциации для удобных вам приложений.
  • Использование для открытия файлов с FTP-серверов и WFX-плагинов.
    Больше не нужно предварительно копировать архив на локальный диск для того, чтобы просмотреть или извлечь его содержимое. Просто настройте ассоциации для удобного вам архиватора.
  • Работа с выделенными файлами.
    Нужно открыть группу файлов в отдельных вкладках текстового редактора или отправить их на печать одним движением? Просто выберите подходящее приложение и правильно настройте ассоциации и параметры командной строки.
  • Ассоциация приложения с папками.
    Хотите открыть в проигрывателе сезон любимого сериала целиком? Убедитесь, что ваш проигрыватель умеет работать с папками и настройте ассоциации.
  • Запуск приложений для уже открытого в Lister файла.
    Просмотрели файл в Lister и поняли, что нужно редактировать? Просто назначьте горячую клавишу для запуска ассоциированного приложения и не забудьте ею воспользоваться :)
  • Копирование искомого текста в буфер.
    Уже оценили по достоинству все возможности плагина и по полной используете его в диалоге «Поиск файлов»? Тогда эта функция для вас. Если вы искали файлы с использованием опции «С текстом», плагин автоматически скопирует искомый текст в буфер, чтобы вы его потом могли вставить уже в диалоге поиска ассоциированного приложения.
Ограничения
  • Работоспособность всех заявленных функций может быть гарантирована только для финальных версий Total Commander 8.0, 8.01, бета-версии 8.5b15. Для предыдущих версий вероятна значительная потеря функционала, для последующих – потеря функционала также не исключается.
  • Запуск ассоциированных приложений будет выполняться вне окна Lister (как при обычном, так и при быстром просмотре).
  • При срабатывании плагина в архиве/FTP-сервере/WFX-плагине, в связи с особенностями поведения Total Commander, выделение не может быть обработано целиком и будет обработан только первый выделенный файл.
Настройка
Настройка плагина производится путём правки файла AppLoader.ini, который должен находиться в папке установки плагина. Создайте этот файл вручную или скопируйте из папки IniSample. Если этот файл не будет найден при загрузке плагина в память, он будет скопирован из папки IniSample автоматически.
Далее приведены ключи файла настроек с описаниями. Значение ключа, указанное после знака '=', является значением по умолчанию, т.е. значением, которое будет использоваться, если ключ отсутствует в файле.

Примечания по именам файлов: Если имя приложения является значением, а не ключом (т.е. находится по правую сторону от знака '='), то в имени можно:
  • использовать переменные окружения типа %COMMANDER_PATH%
  • использовать относительные пути
  • указывать имя файла без пути, если исполняемый файл приложения находится в каталоге плагина, системном каталоге, каталоге Windows или в каталоге, определенном в системной переменной PATH
Примечания по расширениям файлов: Помимо обычных расширений также можно использовать специальные расширения:
  • расширение :dir для папок (например, htm,html,:dir,xml=C:\Program Files\Opera\opera.exe)
  • пустое расширение (например, txt,,ini=notepad, ,txt,ini=notepad или =notepad, но не txt,ini,=notepad)
Примечание по параметрам командной строки:
  • При использовании в параметрах имён файлов имеют место примечания, указанные выше.
  • Используйте специальный символ %1 в параметрах командной строки (ключи секции Editor, ключ DefaultEditor_params секции Options), чтобы указать, в какое место должно быть помещено имя (имена) файла(ов).
    Пример: Для файла <файл>, загружаемого в приложение <приложение>, определены следующие параметры командной строки: <параметры_1> %1 <параметры_2>.
    В результате будет выполнена следующая команда: <приложение> <параметры_1> <файл> <параметры_2>.



Пояснения по работе ключей ParamsMode и ParamsByExtMode.
При срабатывании плагина для одного файла (т.е. из модальных диалогов, Lister или для файла под курсором в панели Total Commander) будет выполнен запуск только одного приложения. При этом, если ParamsMode=1, то при каждом из значений ParamsByExtMode передача приложению параметров командной строки будет выполняться одинаково – в порядке исключения для случая с одним файлом.
Если же срабатывание происходит из панели Total Commander, при том, что выделено более одного файла, то существует несколько вариантов формирования команд для запуска одного или нескольких приложений. Данный пример иллюстрирует работу плагина при различных значениях ключей ParamsMode и ParamsByExtMode.
В активной панели в папке D:\files\ выделены следующие файлы:
file0.ext0
file1.ext0
file0.ext1
file1.ext1
file0.ext2
В секции Editor прописаны следующие строки:
[Editor]
ext1=C:\folder\dir\app0.exe
ext0,ext2=D:\path\app1.exe
ext1_params=-p1 -p2
ext0,ext2_params=/key0
ext0_params=/key1
C:\folder\dir\app0.exe=-p3
D:\path\app1.exe=/key3
Приложения будут запущены таким образом:
1. При ParamsMode=0
"D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" "D:\files\file0.ext2" /key3
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" "D:\files\file1.ext1" -p3
2. При ParamsMode=1, ParamsByExtMode=0
"D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" "D:\files\file0.ext2" /key1
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" "D:\files\file1.ext1" -p1 -p2
3. При ParamsMode=1, ParamsByExtMode=1
"D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" /key1
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" "D:\files\file1.ext1" -p1 -p2
"D:\path\app1.exe" "D:\files\file0.ext2" /key0
4. При ParamsMode=1, ParamsByExtMode=2
"D:\path\app1.exe" "D:\files\file0.ext0" /key1
"D:\path\app1.exe" "D:\files\file1.ext0" /key1
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" -p1 -p2
"C:\folder\dir\app0.exe" "D:\files\file1.ext1" -p1 -p2
"D:\path\app1.exe" "D:\files\file0.ext2" /key0
Имейте в виду, что если приложению при запуске передаются несколько файлов (например, "D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" "D:\files\file0.ext2"), то все они будут открыты в нём только в том случае, если приложение является многодокументным. В противном случае в приложении будет открыт только первый файл.

Настройки Total Commander
Поведение AppLoader в некоторых случаях может зависеть от настроек Total Commander, выполненных через его конфигурационный диалог или конфигурационный файл.

Диалог настроек
Файл настроек
Описание
Страница «Правка/Просмотр», опции в группе «Просмотр по F3» Секция [Configuration], ключ Viewertype Используется плагином для определения поведения при наличии выделения в файловых панелях Total Commander.
Страница «Правка/Просмотр», кнопка «По типам файлов...» Секция [Extensions] Может быть использована плагином для замены секции Editor собственного файла настроек.
  Секция [Configuration], ключ GetTextMode Используется плагином для определения формата, в котором Total Commander будет возвращать список выделенных файлов. При значении ключа 0 плагин не сможет работать с выделенными файлами, имена которых содержат пробелы.

Полное описание данных настроек можно найти в справочном файле Total Commander.
