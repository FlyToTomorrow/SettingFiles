Опис
Плаґін дозволяє відкривати вибрані файли за допомогою асоційованих застосунків у тих випадках, коли є можливість скористатися внутрішнім переглядачем (Lister).
Головною особливістю цього плаґіна є здатність відкрити файл в будь-якому застосунку з таких модальних вікон Total Commander як «Пошук файлів» та «Синхронізація каталогів», а також в Діалозі підтвердження перезапису, чого не можна здійснити ніякими іншими засобами.
Не менш приємною є надавана плаґіном можливість запуску асоційованого застосунку за допомогою гарячої клавіші для вже відкритого у внутрішньому переглядачі файлу.
Окрім того плаґін може працювати з виділеними файлами, має можливості гнучкого налаштування умов спрацьовування, а також функцію вимкнення клавішею Scroll Lock.

Встановлення
Автоматичне
  1. Увійдіть до архіву з плаґіном (Ctrl+PgDn) і Total Commander запропонує встановити його
  2. Дотримуйтеся вказівок Total Commander
Ручне
  1. Розпакуйте вміст архіву до окремого каталогу
  2. Підключіть плаґін:
    • {Конфігурація → Налаштування... → Плаґіни → Плаґіни файлової системи → Налаштування}
    • Додайте AppLoader.wlx (AppLoader.wlx64) до списку плаґінів
  3. Розташуйте AppLoader на першому місці у списку плаґінів
Приклади зручного використання
  • Використання в модальних вікнах.
    Більше не потрібно запускати іншу копію Total Commander, якщо виникає потреба, наприклад, відкрити документ в Microsoft Word або відтворити відеоролик за допомогою зручного плеєра в діалогах «Пошук файлів», «Синхронізація каталогів» або Діалозі підтвердження перезапису. Просто налаштуйте асоціації для зручних вам застосунків.
  • Використання для відкриття файлів з FTP-серверів та WFX-плаґінів.
    Більше не потрібно попередньо копіювати архів на локальний диск для того, щоб переглянути або видобути його вміст. Просто налаштуйте асоціації для зручного вам архіватора.
  • Робота з виділеними файлами.
    Потрібно відкрити групу файлов в окремих вкладках текстового редактора або роздрукувати їх одним рухом? Просто виберіть підходящий застосунок і правильно налаштуйте асоціації та параметри командного рядка.
  • Асоціація застосунка з папками.
    Хочете відкрити в програвачі увесь сезон улюбленого серіалу? Упевніться, що ваш програвач вміє працювати з папками і налаштуйте асоціації.
  • Запуск застосунків для вже відкритого в Lister файлу.
    Переглянули файл в Lister і зрозуміли, що потрібно його редагувати? Просто призначте гарячу клавішу для запуску асоційованого застосунку і не забудьте нею скористатися :)
  • Копіювання шуканого тексту до буфера.
    Вже оцінили належним чином всі можливості плаґіна і на повну використовуєте його в діалозі «Пошук файлів»? Тоді ця функція для вас. Якщо ви шукали файли з використанням опції «З текстом», плаґін автоматично скопіює шуканий текст до буфера, щоб ви його потім могли вставити вже у діалозі пошуку асоційованого застосунка.
Обмеження
  • Працездатність всіх заявлених функцій може бути гарантована тільки для фінальних версій Total Commander 8.0, 8.01, бета-версії 8.5b15. Для попередніх версій ймовірна значна втрата функціоналу, для наступних – втрата функціоналу також не виключається.
  • Запуск асоційованих застосунків буде виконуватися поза вікном Lister (як при звичайному, так і при швидкому перегляді).
  • При спрацьовуванні плаґіна в архіві/FTP-сервері/WFX-плаґіні, у зв'язку з особливостями поведінки Total Commander, виділення не може бути опрацьоване повністю і буде опрацьовано тільки перший виділений файл.
Налаштування
Налаштування плаґіна виконується шляхом редагування файлу AppLoader.ini, який повинен знаходитися в папці встановлення плаґіна. Створіть цей файл вручну або скопіюйте з папки IniSample. Якщо цей файл не буде знайдено при завантаженні плаґіна у пам'ять, його буде скопійовано з папки IniSample автоматично.
Далі приведено ключі файлу налаштувань з описами. Значення ключа, вказане після знака '=', є значенням за замовчуванням, тобто значенням, яке буде використовуватися, якщо ключ відсутній у файлі.

Примітки щодо імен файлів: Якщо ім'я застосунку є значенням, а не ключем (тобто знаходиться праворуч від знака '='), то в імені можна:
  • використовувати змінні оточення типу %COMMANDER_PATH%
  • використовувати відносні шляхи
  • вказувати ім'я файлу без шляху, якщо виконуваний файл застосунка розташований в каталозі плаґіна, системному каталозі, каталозі Windows або в каталозі, визначеному в системній змінній PATH
Примітки щодо розширень файлів: Окрім звичайних розширень також можна використовувати спеціальні розширення:
  • розширення :dir для папок (наприклад, htm,html,:dir,xml=C:\Program Files\Opera\opera.exe)
  • пусте розширення (наприклад, txt,,ini=notepad, ,txt,ini=notepad або =notepad, але не txt,ini,=notepad)
Примітки щодо параметрів командного рядка:
  • При використанні в параметрах імен файлів чинні також примітки, вказані вище.
  • Використовуйте спеціальний символ %1 в параметрах командного рядка (ключі секції Editor, ключ DefaultEditor_params секції Options), щоб вказати, куди саме потрібно помістити ім'я (імена) файлу(ів)
    Приклад: Для файлу <файл>, що завантажується до застосунку <застосунок>, визначено наступні параметри командного рядка: <параметри_1> %1 <параметри_2>.
    В результаті буде виконано наступну команду: <застосунок> <параметри_1> <файл> <параметри_2>.



Пояснення з роботи ключів ParamsMode та ParamsByExtMode.
При спрацюванні плаґіна для одного файлу (тобто з модальних діалогів, Lister або для файлу під курсором в панелі Total Commander) буде виконано запуск тільки одного застосунка. При цьому, якщо ParamsMode=1, то при кожному із значень ParamsByExtMode передавання застосунку параметрів командного рядка буде виконуватися однаково – як виняток для випадку з одним файлом.
Якщо ж спрацьовування відбувається з панелі Total Commander, тоді як виділено більше одного файлу, то існує декілька варіантів формування команд для запуску одного або декількох застосунків. Даний приклад ілюструє роботу плаґіна при різних значеннях ключів ParamsMode та ParamsByExtMode.
В активній панелі в папці D:\files\ виділено наступні файли:
file0.ext0
file1.ext0
file0.ext1
file1.ext1
file0.ext2
В секції Editor прописано наступні рядки:
[Editor]
ext1=C:\folder\dir\app0.exe
ext0,ext2=D:\path\app1.exe
ext1_params=-p1 -p2
ext0,ext2_params=/key0
ext0_params=/key1
C:\folder\dir\app0.exe=-p3
D:\path\app1.exe=/key3
Застосунки буде запущено таким чином:
1. При ParamsMode=0
"D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" "D:\files\file0.ext2" /key3
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" "D:\files\file1.ext1" -p3
2. При ParamsMode=1, ParamsByExtMode=0
"D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" "D:\files\file0.ext2" /key1
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" "D:\files\file1.ext1" -p1 -p2
3. При ParamsMode=1, ParamsByExtMode=1
"D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" /key1
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" "D:\files\file1.ext1" -p1 -p2
"D:\path\app1.exe" "D:\files\file0.ext2" /key0
4. При ParamsMode=1, ParamsByExtMode=2
"D:\path\app1.exe" "D:\files\file0.ext0" /key1
"D:\path\app1.exe" "D:\files\file1.ext0" /key1
"C:\folder\dir\app0.exe" "D:\files\file0.ext1" -p1 -p2
"C:\folder\dir\app0.exe" "D:\files\file1.ext1" -p1 -p2
"D:\path\app1.exe" "D:\files\file0.ext2" /key0
Майте на увазі, якщо застосунку при запуску передаються декілька файлів (наприклад, "D:\path\app1.exe" "D:\files\file0.ext0" "D:\files\file1.ext0" "D:\files\file0.ext2"), то всі вони будуть відкриті в ньому тільки у тому випадку, якщо застосунок багатодокументний. У протилежному випадку в застосунку буде відкрито тільки перший файл.

Налаштування Total Commander
Поведінка AppLoader в деяких випадках може залежати від налаштувань Total Commander, виконаних через його конфігураційний діалог чи конфігураційний файл.

Діалог налаштувань
Файл налаштувань
Опис
Сторінка «Правка/Перегляд», опції в групі «Перегляд за F3» Секція [Configuration], ключ Viewertype Використовується плаґіном для визначення поведінки за наявності виділення в файлових панелях Total Commander.
Сторінка «Правка/Перегляд», кнопка «За типами файлів...» Секція [Extensions] Може бути використана плаґіном для заміни секції Editor власного файлу налаштувань.
  Секція [Configuration], ключ GetTextMode Використовується плаґіном для визначення формату, в якому Total Commander буде повертати список виділених файлів. При значенні ключа 0 плаґін не зможе працювати з виділеними файлами, імена яких містять пробіли.

Повний опис цих налаштувань можна знайти у довідковому файлі Total Commander.
